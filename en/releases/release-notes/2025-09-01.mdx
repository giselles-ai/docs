This update brings improvements to GitHub Trigger configuration, enables multi-embedding by default, and includes significant internal architecture updates for better performance and stability.

## GitHub Trigger Improvements

**GitHub Trigger Node** now supports more flexible configuration:

- **Label-based triggering**: Trigger workflows based on specific labels
- **Repository modification**: Change connected repositories after initial configuration
- **Improved credential handling**: Update GitHub credentials without resetting tool selections

## Multi-Embedding Now Default

The **multi-embedding feature** is now enabled by default for all users. This allows you to use multiple embedding models simultaneously for better search accuracy across your knowledge bases.

## Enhanced Error Handling

- Unified error tracing across the generation pipeline
- Better error logging for failed AI SDK generations
- Improved failure details display including error messages

## UI Improvements

- Redesigned sidebar with navigation rail for easier access
- Toast notifications now auto-dismiss for cleaner experience
- Enhanced properties panel layout
- Unified text generation select components

## Bug Fixes

- Fixed workspace access for all user's teams
- Fixed multi-image paste upload and duplicate handling
- Resolved default filter alignment issues
