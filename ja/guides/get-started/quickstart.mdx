## Giselle で初めての AI ワークフローを構築しよう

Giselle を数分で使い始めましょう！このクイックスタートガイドでは、サインアップから完全なワークフローの実行まで、初めての Workspace の作成をステップバイステップで説明します。ユーザーフレンドリーなインターフェースでワークフローの構築とテストの方法を学びましょう。

<Steps>
  <Step title="サインアップ">
    Giselle の力を解き放ちましょう！初めてご利用の方は、[サインアップ](https://studio.giselles.ai/signup)ページでアカウントを作成してください。
    すでに Giselle コミュニティの一員の方は、[ログイン](https://studio.giselles.ai/login)してすぐに始めましょう。
  </Step>
  <Step title="Workspace を構築する">
    <AccordionGroup>
      <Accordion title="Workspace を作成する" defaultOpen={true}>
        [Workspaces](https://studio.giselles.ai/apps) ページに移動し、右上の「New Workspace」ボタンをクリックして初めての Workspace の作成を開始します。
      </Accordion>
      <Accordion title="ノードを配置する" defaultOpen={true}>
        下部のツールバーから「Generator」ノードをキャンバスにドラッグ＆ドロップして配置します。これがワークフローの最初のノードになります。
      </Accordion>
      <Accordion title="Prompt を設定する" defaultOpen={true}>
        右側のパネルで「Prompt」タブを開き、テキストエリアにプロンプトを入力します。このテキストがノードの動作を導きます。
        ```markdown
        動物に関する興味深いトピックを1つ提案してください。
        ```
      </Accordion>
      <Accordion title="Model を選択する" defaultOpen={true}>
        「Model」タブで、使用する AI モデルのプロバイダー（OpenAI、Anthropic、Google など）とモデルを選択します。
      </Accordion>
      <Accordion title="ノードをテストする" defaultOpen={true}>
        右パネルの上部にある「Generate」ボタン（または Cmd+Enter）をクリックします。これによりこのノードのみが実行され、プロンプトをテストして出力を確認できます。
      </Accordion>
    </AccordionGroup>
  </Step>
  <Step title="ワークフローとして実行する">
    <AccordionGroup>
      <Accordion title="別のノードを追加する" defaultOpen={true}>
        ワークフローを構築するために、別のノードを追加しましょう。下部のツールバーから「Generator」ノードをドラッグして、キャンバスに配置します。
      </Accordion>
      <Accordion title="ノードを接続する" defaultOpen={true}>
        2つのノードを接続するには、最初のノードの Output ポートから2番目のノードの Input ポートへドラッグしてつなぎます。
      </Accordion>
      <Accordion title="2番目のノードの Prompt を設定する" defaultOpen={true}>
        2番目のノードの「Prompt」タブに以下を入力します。最初のノードの出力が2番目のノードに入力されることで、連続したワークフローが構築されます。
        ```markdown
        以下のトピックについて調査してください：
        ```
      </Accordion>
      <Accordion title="ワークフローを実行する" defaultOpen={true}>
        最後に、画面右上の「Run」ボタンをクリックしてワークフローを実行します。これにより両方のノードが順番に実行され、最終出力が生成されます。おめでとうございます！Giselle で初めてのワークフローを実行しました！
      </Accordion>
    </AccordionGroup>
  </Step>
</Steps>
